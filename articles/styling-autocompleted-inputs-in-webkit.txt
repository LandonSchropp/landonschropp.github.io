<p>You know you hate it: that ugly yellow background color in Chrome and Safari’s autocompleted fields. This color is set in the user agent stylesheet, s0 unfortunately there’s no way to override it.</p>
<p>But there’s hope! I found a neat little trick in this <a href="http://stackoverflow.com/a/13860010/262125">Stack Overflow answer</a>. Instead of styling the background color, use an inner box shadow that covers the background and use -webkit-text-fill-color to set the text color.</p>
<pre><code class="language-css"><span class="hljs-selector-tag">input</span>:-webkit-autofill,
input:-webkit-autofill:focus {
  <span class="hljs-attribute">box-shadow</span>: <span class="hljs-number">0</span> <span class="hljs-number">0</span> <span class="hljs-number">0</span> <span class="hljs-number">1000px</span> white inset;
  -webkit-<span class="hljs-selector-tag">text</span>-<span class="hljs-attribute">fill</span>-<span class="hljs-attribute">color</span>: <span class="hljs-number">#333</span>;
}
</code></pre>
<p>Now you can style these inputs however you want! However, the background color is yellow for a reason—it lets people know an input has been filled in by the browser. Instead of getting rid of it, consider choosing a different color that more closely matches your site’s design.</p>
